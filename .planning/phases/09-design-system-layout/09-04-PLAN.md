  <name>Task 1: Create InsetHero component</name>
  <files>src/components/layout/InsetHero.tsx</files>
  <action>
    Create a full-width hero component with inset rounded corners, background image, dark overlay, and content area.

    **Props interface:**
    ```ts
    interface InsetHeroProps {
      backgroundImage: string;   // image path for next/image
      children: React.ReactNode; // content to overlay (heading, subheading, CTA)
      height?: string;           // CSS min-height, default "70vh"
      className?: string;
      overlayClassName?: string; // custom overlay gradient classes
      priority?: boolean;        // next/image priority loading, default true
    }
    ```

    **Rendering:**
    ```tsx
    <section className={cn('px-4 pt-4', className)}>
      <div
        className="relative overflow-hidden rounded-[20px]"
        style={{ minHeight: height }}
      >
        {/* Background image */}
        <Image
          src={backgroundImage}
          alt=""
          fill
          className="object-cover"
          priority={priority}
        />
        {/* Dark overlay gradient */}
        <div className={cn(
          'absolute inset-0 bg-gradient-to-r from-primary-950/80 via-primary-950/50 to-primary-950/30',
          overlayClassName
        )} />
        {/* Content */}
        <div className="relative z-10 flex h-full items-end">
          <div className="mx-auto w-full max-w-7xl px-6 pb-16 pt-32 md:pb-20 md:pt-40">
            {children}
          </div>
        </div>
      </div>
    </section>
    ```

    **Key details:**
    - The hero is INSET from page edges: `px-4 pt-4` creates the ~16px margin around the image card
    - Rounded corners: `rounded-[20px]` for the card-like feel (~20-24px from reference)
    - The Image alt="" because the image is decorative (content is overlaid text)
    - Default gradient goes left-to-right (darker on left where heading typically sits, lighter on right)
    - Content is positioned at the bottom of the hero (items-end + pb-16/20)
    - Large top padding (pt-32/40) accounts for the transparent header overlaying the hero
    - The `height` prop defaults to "70vh" for homepage, interior pages will pass "50vh"
    - `priority={true}` by default since hero images should load immediately (LCP)
    - SERVER component (no "use client")
    - Import Image from "next/image"
    - The `children` pattern allows each page to compose its own hero content (homepage: two-column with heading + CTA, interior pages: just a centered heading)
  </action>

<task type="auto">
  <name>Task 3: Update Header with transparent/solid mode and scroll transition</name>
  <files>src/components/layout/Header.tsx</files>
  <action>
    The v2.0 design has the header overlaying the hero image with a transparent background. The header needs two modes and a scroll-based transition.

    **New prop:**
    ```ts
    interface HeaderProps {
      variant?: 'solid' | 'transparent';  // default: 'solid'
    }
    ```

    **Behavior:**
    - `variant="solid"` (DEFAULT): Current behavior -- sticky, `bg-surface-page` (was `bg-stone-50`), dark text, border-b. Used on pages WITHOUT a full hero (or as fallback).
    - `variant="transparent"`: Fixed at top, transparent background, white text/logo. Used on pages WITH InsetHero. Transitions to solid background after scrolling ~100px.

    **Implementation:**
    Since the header is already "use client" (uses useState for MobileNav), we can add scroll detection:

    ```tsx
    const [scrolled, setScrolled] = useState(false);

    useEffect(() => {
      if (variant !== 'transparent') return;

      const handleScroll = () => {
        setScrolled(window.scrollY > 100);
      };

      window.addEventListener('scroll', handleScroll, { passive: true });
      return () => window.removeEventListener('scroll', handleScroll);
    }, [variant]);
    ```

    **Styling logic:**
    ```tsx
    const isTransparent = variant === 'transparent' && !scrolled;

    <header className={cn(
      variant === 'transparent' ? 'fixed' : 'sticky',
      'top-0 z-50 w-full transition-colors duration-300',
      isTransparent
        ? 'bg-transparent'
        : 'bg-surface-page/95 backdrop-blur-sm border-b border-divider'
    )}>
    ```

    **Text/link color changes:**
    - Logo: `isTransparent ? 'text-white' : 'text-primary-900'`
    - Nav links: `isTransparent ? 'text-white/90 hover:text-white' : 'text-text-body hover:text-primary-900'`
    - Active link underline: Change from `after:bg-accent-500` to `after:bg-accent-400` (brighter blue, visible on both light and dark backgrounds)
    - Mobile hamburger: `isTransparent ? 'text-white' : 'text-text-body'`

    **Other updates (v2.0 tokens):**
    - Replace `bg-stone-50` with `bg-surface-page` (or `bg-surface-page/95` with `backdrop-blur-sm` for a modern frosted effect)
    - Replace `border-stone-200` with `border-divider`
    - The MobileNav component does NOT need changes here (it's a separate file, and its bg-stone-50 will auto-update to the new stone-50 token value from Plan 01)

    **Important:** The Header receives `variant` as a prop. This means the ROOT LAYOUT needs to be able to pass it. However, layout.tsx is a server component and Header is a client component. The prop approach works fine -- pages that need transparent header will need a different layout approach.

    Actually, a cleaner approach for the layout: Since layout.tsx renders Header statically, and different pages need different header modes, the header should read its mode from a React context or from a CSS-based approach. BUT the simplest approach for now: export the Header with a default `variant="solid"` and let the homepage (and other hero pages) override it by NOT using the layout's header and instead rendering their own Header with `variant="transparent"`.

    WAIT -- that's too complex for Phase 9. Simpler approach: Just add the prop and the scroll logic. The actual page-level wiring (how homepage passes variant="transparent") will happen in Phase 10 when the homepage is redesigned. For now, the Header defaults to 'solid' and works exactly as before but with updated token colors (stone-50 -> surface-page, stone-200 -> divider).

    So for THIS task:
    1. Add the `variant` prop with default 'solid'
    2. Add scroll detection logic (only active when variant='transparent')
    3. Add conditional styling for transparent vs solid
    4. Update all v1.0 stone-* references to v2.0 surface/divider tokens
    5. The Header continues to work as-is in the current layout (solid mode)
    6. Phase 10 will wire up variant="transparent" for the homepage
  </action>
