  <name>Task 1: Create EmphasisHeading component</name>
  <files>src/components/shared/EmphasisHeading.tsx</files>
  <action>
    Create a React component that parses text with markdown-style `*italic*` markers and renders `<em>` elements with the heading font's italic style.

    **Props interface:**
    ```ts
    interface EmphasisHeadingProps {
      text: string;           // "Support That *Strengthens* Rural Communities"
      as?: 'h1' | 'h2' | 'h3' | 'h4';  // default: 'h2'
      className?: string;
      accentItalic?: boolean; // when true, italic words also get text-accent-500 color (default: false)
    }
    ```

    **Parsing logic:**
    - Split text on `(\*[^*]+\*)` regex pattern (captures *italic* groups)
    - For parts that start and end with *, render: `<em key={i} className={cn("italic font-heading", accentItalic && "text-accent-500")}>{part.slice(1, -1)}</em>`
    - For other parts, render as plain text nodes
    - The `<em>` tag provides semantic emphasis AND triggers CSS italic

    **Heading tag:**
    - Render using the dynamic `Tag` pattern: `const Tag = as`
    - Apply `font-heading` to the Tag className (ensures Cormorant Garamond on the whole heading)
    - The italic `<em>` inherits the serif font and just adds italic style

    **Important details:**
    - This must be a SERVER component (no "use client") -- it does pure rendering, no state/effects
    - The `font-heading` class maps to `var(--font-cormorant-garamond)` via globals.css @theme
    - Do NOT hardcode font-family -- use Tailwind's font-heading utility
    - The component must handle edge cases: no asterisks (renders plain text), multiple italic groups, adjacent italic groups, asterisks at start/end of string
    - Use `cn()` from `@/lib/utils` for className merging
  </action>

<task type="auto">
  <name>Task 3: Create CircleArrowCTA component</name>
  <files>src/components/shared/CircleArrowCTA.tsx</files>
  <action>
    Create a CTA component that renders text + circular arrow icon as one clickable unit.

    **Props interface:**
    ```ts
    interface CircleArrowCTAProps {
      label: string;              // "Book a Consultation", "Get in Touch"
      href?: string;              // if provided, renders as Link; otherwise as button
      variant?: 'dark' | 'light'; // dark = navy circle white arrow (on light bg), light = white circle navy arrow (on dark/image bg)
      className?: string;
      onClick?: () => void;
      type?: 'button' | 'submit'; // for button mode, default 'button'
    }
    ```

    **Rendering pattern:**
    The component renders text label + filled circle with ArrowUpRight icon side by side.

    ```tsx
    const Wrapper = href ? Link : 'button';
    const wrapperProps = href ? { href } : { type: type || 'button', onClick };

    return (
      <Wrapper {...wrapperProps} className={cn('group inline-flex items-center gap-3', className)}>
        <span className={cn(
          'text-base font-medium font-body',
          variant === 'dark' ? 'text-primary-900' : 'text-white'
        )}>
          {label}
        </span>
        <span className={cn(
          'flex h-10 w-10 items-center justify-center rounded-full transition-transform duration-300 group-hover:scale-110',
          variant === 'dark'
            ? 'bg-primary-900 text-white'
            : 'bg-white text-primary-900'
        )}>
          <ArrowUpRight className="h-5 w-5" aria-hidden="true" />
        </span>
      </Wrapper>
    );
    ```

    **Important details:**
    - Import `Link` from `next/link` and `ArrowUpRight` from `lucide-react`
    - This must be a SERVER component (no "use client") -- Link and static rendering only, no state
    - The circle hover scales up slightly (scale-110) on the `group-hover` -- this is the primary interaction feedback
    - Default variant is 'dark' (navy circle, for use on light backgrounds)
    - The ArrowUpRight icon must have `aria-hidden="true"` (the label text provides the accessible name)
    - Circle size: h-10 w-10 (40px) -- matches the ~40-44px from the design reference
    - The whole unit (text + circle) is the click target, not just the circle
    - Use `cn()` for className merging
    - If both href and onClick are provided, href wins (renders as Link)
  </action>
