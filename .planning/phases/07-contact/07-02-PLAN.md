    2. Create `src/components/contact/ContactForm.tsx`:

    This is a `"use client"` component because it needs:
    - `react-hook-form` for form state management
    - `useActionState` from React for server action integration
    - `useRef` and `useEffect` for form reset on success

    **Component structure:**

    ```tsx
    "use client";

    import { useActionState, useRef, useEffect, useState } from "react";
    import { useForm } from "react-hook-form";
    import { zodResolver } from "@hookform/resolvers/zod";
    import { contactSchema, type ContactFormData, type ContactFormState, AUDIENCE_OPTIONS } from "@/lib/schemas/contact";
    import { submitContact } from "@/app/contact/_actions/submit-contact";
    ```

    **State management approach:**
    - Use `useActionState` (React 19) for the server action, NOT `useFormState` from `react-dom` (deprecated)
    - Use react-hook-form for client-side validation ONLY
    - On form submit: run client validation first via react-hook-form, then submit to server action via the form's native action
    - Wire them together: react-hook-form's `handleSubmit` validates, then calls `formRef.current.requestSubmit()` to trigger the server action

    **Implementation details:**

    ```tsx
    const initialState: ContactFormState = {
      success: false,
      message: "",
    };

    export function ContactForm() {
      const [state, formAction, isPending] = useActionState(submitContact, initialState);
      const formRef = useRef<HTMLFormElement>(null);
      const [showSuccess, setShowSuccess] = useState(false);

      const {
        register,
        handleSubmit,
        formState: { errors },
        reset,
      } = useForm<ContactFormData>({
        resolver: zodResolver(contactSchema),
        defaultValues: {
          name: "",
          email: "",
          phone: "",
          audience: "",
          message: "",
          honeypot: "",
        },
      });

      // Handle server action response
      useEffect(() => {
        if (state.success && state.message) {
          setShowSuccess(true);
          reset();
        }
      }, [state, reset]);

      // Client-side validation then server submission
      const onSubmit = () => {
        formRef.current?.requestSubmit();
      };

      const handleSendAnother = () => {
        setShowSuccess(false);
      };
    ```

    **Form JSX structure:**

    If `showSuccess` is true, render the success state:
    ```tsx
    <div className="rounded-lg border border-success/30 bg-success/10 p-8 text-center">
      <div className="mx-auto mb-4 flex h-12 w-12 items-center justify-center rounded-full bg-success/20">
        <CheckCircle2 className="h-6 w-6 text-success" />
      </div>
      <h3 className="font-heading text-2xl font-semibold text-text-primary">
        Thank you for getting in touch
      </h3>
      <p className="mt-2 font-body text-text-body">
        We'll get back to you as soon as we can.
      </p>
      <button
        type="button"
        onClick={handleSendAnother}
        className="mt-6 font-body text-sm font-medium text-accent-600 underline underline-offset-4 transition-colors hover:text-accent-500"
      >
        Send another message
      </button>
    </div>
    ```
    Import `CheckCircle2` from `lucide-react`.

    Otherwise render the form:

    ```tsx
    <form
      ref={formRef}
      action={formAction}
      onSubmit={(e) => {
        e.preventDefault();
        handleSubmit(onSubmit)(e);
      }}
      className="space-y-6"
      noValidate
    >
    ```

    **Error summary at top** (only if server errors or client errors exist):
    ```tsx
    {(state.errors || Object.keys(errors).length > 0) && (
      <div
        className="rounded-lg border border-error/30 bg-error/10 p-4"
        role="alert"
        aria-live="polite"
      >
        <p className="font-body text-sm font-medium text-error">
          Please check the form below and try again.
        </p>
      </div>
    )}
    ```

    **Form fields -- each field follows this pattern:**

    ```tsx
    <div>
      <label
        htmlFor="name"
        className="block font-body text-sm font-medium text-text-primary"
      >
        Name <span className="text-error">*</span>
      </label>
      <input
        {...register("name")}
        type="text"
        id="name"
        autoComplete="name"
        aria-invalid={!!errors.name}
        aria-describedby={errors.name ? "name-error" : undefined}
        className="mt-1 block w-full rounded-md border border-stone-300 bg-stone-300 px-4 py-3 font-body text-base text-text-primary placeholder:text-text-muted/60 focus:border-accent-500 focus:bg-stone-50 focus:outline-none focus:ring-2 focus:ring-accent-500/30 transition-colors"
        placeholder="Your name"
      />
      {errors.name && (
        <p id="name-error" className="mt-1.5 font-body text-sm text-error" role="alert">
          {errors.name.message}
        </p>
      )}
    </div>
    ```

    Apply this pattern to all fields:

    **Name field:** type="text", autoComplete="name", required indicator
    **Email field:** type="email", autoComplete="email", required indicator
    **Phone field:** type="tel", autoComplete="tel", NO required indicator, add helper text "Optional" in lighter color after the label
    **Audience selector:** `<select>` element with `register("audience")`, autoComplete="off", NO required indicator, add helper text "Optional". Default `<option value="">Select if you'd like...</option>` followed by `AUDIENCE_OPTIONS.map()`. Same styling as inputs but add `appearance-none` and custom chevron via background-image or a wrapper.
    **Message field:** `<textarea>`, rows={5}, autoComplete="off", required indicator
    **Honeypot field:** Hidden from real users:
    ```tsx
    <div
      className="absolute left-[-9999px]"
      aria-hidden="true"
    >
      <label htmlFor="honeypot">Leave this empty</label>
      <input
        {...register("honeypot")}
        type="text"
        id="honeypot"
        tabIndex={-1}
        autoComplete="off"
      />
    </div>
    ```

    **Reassurance text above submit:**
    ```tsx
    <p className="font-body text-sm text-text-muted">
      Your information is confidential. We will only use it to respond to your message.
    </p>
    ```

    **Submit button:**
    ```tsx
    <button
      type="submit"
      disabled={isPending}
      className="w-full rounded-md bg-primary-900 px-8 py-3.5 font-body text-base font-medium text-text-on-dark transition-all duration-300 ease-premium hover:bg-primary-700 focus:outline-none focus:ring-2 focus:ring-accent-500 focus:ring-offset-2 disabled:cursor-not-allowed disabled:opacity-60"
    >
      {isPending ? "Sending..." : "Send Message"}
    </button>
    ```
    - Button text is "Send Message" (NEVER "Submit" per voice & tone guide)
    - Shows "Sending..." while server action is pending
    - Full width, navy background, gold focus ring
    - disabled state with reduced opacity while pending

    **Important implementation notes:**
    - The `noValidate` on the form element disables browser-native validation (we use zod instead)
    - `aria-invalid` and `aria-describedby` on inputs link errors to fields for screen readers
    - Never clear fields on error (react-hook-form preserves values by default)
    - The form wraps a `relative` container so the honeypot's `absolute` positioning is relative to the form
    - Add `relative` to the form's className
    - All inputs use `bg-stone-300` background per design spec, switching to `bg-stone-50` on focus
