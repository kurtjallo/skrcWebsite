**AnimatedSection (src/components/shared/AnimatedSection.tsx):**
- `"use client"` -- Motion requires client
- Import `LazyMotion`, `domAnimation`, `m`, `useReducedMotion` from `motion/react` (NOT `framer-motion` -- the package is now called `motion`)
- Accept props:
  ```ts
  interface AnimatedSectionProps {
    children: React.ReactNode;
    className?: string;
    delay?: number;        // default 0
    staggerChildren?: number; // default 0.12 (120ms)
    as?: "div" | "section" | "article"; // default "div"
  }
  ```
- Wrap in `<LazyMotion features={domAnimation}>` for tree-shaking (~15KB saving per PERF-03)
- Use `m.div` (or m[as]) with:
  ```ts
  const variants = {
    hidden: { opacity: 0, y: 30 },
    visible: {
      opacity: 1,
      y: 0,
      transition: {
        duration: 0.6,
        ease: [0.25, 0.1, 0.25, 1],
        delay,
        staggerChildren,
      },
    },
  };

  const childVariants = {
    hidden: { opacity: 0, y: 20 },
    visible: {
      opacity: 1,
      y: 0,
      transition: {
        duration: 0.5,
        ease: [0.25, 0.1, 0.25, 1],
      },
    },
  };
  ```
- Apply `initial="hidden"`, `whileInView="visible"`, `viewport={{ once: true, margin: "-100px" }}`
- Check `useReducedMotion()`: if true, render children without any Motion wrapper (plain div/section/article with className). Do NOT just zero out durations -- skip the Motion component entirely for reduced motion.
- Export both `AnimatedSection` (the wrapper) and `animatedChildVariants` (so children can use `<m.div variants={animatedChildVariants}>` for stagger coordination).

  ```ts
  }
  ```
- Background logic for `variant="auto"` (the default):
  ```ts
  // Alternating rhythm: stone-50, stone-100, stone-50, primary-900, stone-100, repeat
  const AUTO_BACKGROUNDS = [
    "bg-stone-50",    // 0
    "bg-stone-100",   // 1
    "bg-stone-50",    // 2
    "bg-primary-900 text-stone-100", // 3 -- dark dramatic break, flip text color
    "bg-stone-100",   // 4
  ] as const;

  function getAutoBackground(index: number): string {
    return AUTO_BACKGROUNDS[index % AUTO_BACKGROUNDS.length];
  }
  ```
- Manual variants:
  - `"light-a"` -> `bg-stone-50`
  - `"light-b"` -> `bg-stone-100`
  - `"dark"` -> `bg-primary-900 text-stone-100`

  - If `animated`, wrap inner content in `<AnimatedSection>`
  - Pass `id` and additional `className` through

**Button gold sweep hover (src/components/ui/button.tsx):**

Add a new variant `"gold"` to the button variants (alongside default, destructive, outline, etc.):

```tsx
gold: "relative overflow-hidden bg-primary-900 text-stone-50 before:content-[''] before:absolute before:inset-0 before:bg-accent-500 before:translate-x-[-100%] before:transition-transform before:duration-400 before:ease-premium hover:before:translate-x-0 hover:text-primary-950 before:z-0 [&>*]:relative [&>*]:z-10 inline-flex items-center justify-center font-body font-medium"
```

Key details for the gold sweep:
- Button has `overflow-hidden` and `relative`
- `::before` pseudo starts at `translateX(-100%)` (fully off-screen left)
- On hover, `::before` transitions to `translateX(0)` (sweeps in from left)
- `::before` is gold (`bg-accent-500`), sits behind text
- Text content needs `relative z-10` to stay above the sweep -- handle via `[&>*]:relative [&>*]:z-10` or wrap children in a span

Also add `"gold-outline"` variant:
```tsx
"gold-outline": "relative overflow-hidden border-2 border-accent-500 text-accent-600 bg-transparent before:content-[''] before:absolute before:inset-0 before:bg-accent-500 before:translate-x-[-100%] before:transition-transform before:duration-400 before:ease-premium hover:before:translate-x-0 hover:text-primary-950 before:z-0 [&>*]:relative [&>*]:z-10 inline-flex items-center justify-center font-body font-medium"
```

Make sure `ease-premium` (the cubic-bezier) is used everywhere, NOT `ease-in-out`. The `ease-premium` timing function should be defined in Tailwind config from Phase 1.

Important: Do NOT break existing shadcn variant behavior. The gold variants are additive.
