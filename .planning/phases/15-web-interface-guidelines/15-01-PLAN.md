    Replace every `transition-all` with the narrowest explicit transition that covers only the properties actually being animated. Use the analysis below for each file.

    **Replacement map (each line: file -> what's animated -> replacement):**

    2. `src/components/ui/card.tsx:16` — hoverable cards animate transform + shadow -> replace `transition-all` with `transition-[transform,box-shadow]`

    8. `src/components/home/SelfIdCards.tsx:66` — cards: transform + shadow -> replace `transition-all` with `transition-[transform,box-shadow]`
    9. `src/components/home/GetInvolved.tsx:93` — cards: transform + shadow -> replace `transition-all` with `transition-[transform,box-shadow]`

    18. `src/app/thank-you/page.tsx:68` — phone pill: border-color, bg-color -> replace `transition-all` with `transition-colors`

    **Why explicit transitions matter:** `transition-all` causes the browser to watch every single CSS property for changes (layout, paint, composite), triggering unnecessary style recalculations. Explicit properties let the browser optimize.

    **Tailwind syntax notes:**
    - `transition-colors` is a built-in Tailwind utility (covers color, background-color, border-color, text-decoration-color, fill, stroke)
    - `transition-transform` is a built-in Tailwind utility (covers transform)
    - `transition-[transform,box-shadow]` is an arbitrary value utility for multiple specific properties
    - `transition-[background-color,opacity]` is an arbitrary value utility

    - Footer.tsx: Change `width={160} height={53}` to `width={35} height={53}` (approximate 2:3 ratio; CSS `h-12 w-auto` controls display)

    **WIG-04: Hydration-safe year**

    In `Footer.tsx` (line 128-131), add `suppressHydrationWarning` to the `<p>` element containing `{new Date().getFullYear()}`. This prevents the React hydration warning if the year changes between server pre-render (build time) and client hydration (user visit crossing year boundary).

    Change:
    ```jsx
    <p className="text-center text-xs text-white/50 sm:text-left">
    ```
    To:
    ```jsx
    <p className="text-center text-xs text-white/50 sm:text-left" suppressHydrationWarning>
    ```

    **Why `suppressHydrationWarning` and not hardcoded year:** A hardcoded year would require a code change every January. `suppressHydrationWarning` is the idiomatic React/Next.js approach for dynamic values that may differ between server and client (see Next.js docs on date rendering). The value is always correct — we just tell React not to warn if they differ.
