  <action>
    Create the homepage page file that imports and renders all 9 section components in the correct order.

    **This is a Server Component** -- do NOT add `"use client"`. All interactive behavior lives in the child components (which are client components). The page itself is a pure composition layer.

    **Imports:**
    ```typescript
    import Hero from '@/components/home/Hero';
    import SelfIdCards from '@/components/home/SelfIdCards';
    import ServicePillars from '@/components/home/ServicePillars';
    import ImpactStats from '@/components/home/ImpactStats';
    import FeaturedTestimonial from '@/components/home/FeaturedTestimonial';
    import EventsPreview from '@/components/home/EventsPreview';
    import GetInvolved from '@/components/home/GetInvolved';
    import Newsletter from '@/components/home/Newsletter';
    import PartnerLogos from '@/components/home/PartnerLogos';
    ```

    **Metadata export:**
    ```typescript
    import type { Metadata } from 'next';

    export const metadata: Metadata = {
      title: 'St Katharine Rural Connect | Support, Connection & Care for Rural Communities',
      description: 'St Katharine Rural Connect strengthens rural communities through accessible support, practical resources, and meaningful belonging for farmers, offshore workers, families, and community members.',
    };
    ```

    **Page component:**
    ```typescript
    export default function HomePage() {
      return (
        <main>
          <Hero />
          <SelfIdCards />
          <ServicePillars />
          <ImpactStats />
          <FeaturedTestimonial />
          <EventsPreview />
          <GetInvolved />
          <Newsletter />
          <PartnerLogos />
        </main>
      );
    }
    ```

    The page itself is minimal -- a clean composition of sections. Each section component manages its own:
    - Background color (following the alternating rhythm)
    - Internal padding and max-width
    - Animations and interactivity

    **Alternating section rhythm** (each component handles its own bg):
    ```
    Hero:              gradient (primary-950 to primary-800)
    SelfIdCards:       stone-100
    ServicePillars:    stone-50
    ImpactStats:       primary-900 (dark dramatic break)
    FeaturedTestimonial: stone-100
    EventsPreview:     stone-50
    GetInvolved:       stone-100
    Newsletter:        stone-50
    PartnerLogos:      stone-100
    ```

    If any component does NOT match the rhythm above, update the component's background class to match. Do NOT add wrapper divs in page.tsx for backgrounds -- each component owns its background.

    **Important:** Do NOT wrap sections in extra divs or add spacing between them in page.tsx. Each component should have its own vertical padding (py-16 md:py-24) which creates natural spacing. The `<main>` wrapper is sufficient.

    If the project has a SectionWrapper component from Phase 2, and components are not yet using it, add a comment: `{/* TODO: Migrate section components to use SectionWrapper from Phase 2 */}`.
